# <a name="activating-the-visual-studio-team-services-vsts-benefit-in-visual-studio-subscriptions"></a>Aktivieren der in Ihrem Visual Studio-Abonnement enthaltenen Visual Studio Team Services

Über die meisten Visual Studio-Abonnements können Sie auf Visual Studio Team Services (VSTS) zugreifen.  VSTS ist eine perfekte Ergänzung Ihrer IDE, mit der Git-Repositorys, Agile-Tools für die Planung und gehostete Builds für jede Sprache bereitgestellt werden.  

Um VSTS zu verwenden, benötigen Sie ein Konto. Sie können dafür ein vorhandenes nutzen oder ein neues erstellen.  So erstellen Sie ein neues Konto:
1.  Klicken Sie unten auf der VSTS-Kachel auf den Link „Get Started“ (Los geht‘s).   

![VSTS-Kachel](_img\vs-vsts\vs-vsts-tile.png)

2.  Geben Sie einen Namen für Ihre VSTS-Projektwebsite ein.  
3.  Wählen Sie aus, ob Sie das Projekt mit **Git** oder **Team Foundation-Versionskontrolle (Team Foundation Version Control, TFVC)** verwalten möchten.  Dies ist eine unveränderliche Auswahl für jedes Teamprojekt, das Sie erstellen, aber Sie können TFVC und Git-Teamprojekte in derselben Teamprojektsammlung verwenden.  Sind Sie nicht sicher, wofür Sie sich entscheiden sollen? 
- Git ist ein dezentralisiertes Quellcodeverwaltungssystem. Jeder Entwickler verfügt auf seinem Entwicklungscomputer über eine Kopie des gesamten Quellrepositorys. Entwickler können für alle Änderungen auf ihrem Entwicklungscomputer Commits ausführen und Versionskontrollvorgänge wie den Aufruf von Versionsgeschichten oder Vergleichen ohne Netzwerkverbindung vornehmen.  [Weitere Informationen zu Git](https://www.visualstudio.com/en-us/docs/git/gitquickstart)
- TFVC: Team Foundation-Versionskontrolle ist ein zentralisiertes Versionskontrollsystem. In der Regel verfügen Teammitglieder auf ihren Entwicklungscomputern nur über eine Version jeder Datei. Daten zur Versionsgeschichte einer Datei werden nur auf dem Server gespeichert. Verzweigungen sind pfadbasiert und werden auf dem Server erstellt. [Weitere Informationen zu Team Foundation-Versionskontrolle (TFVC)](https://www.visualstudio.com/en-us/docs/tfvc/overview)
 
4.  Klicken Sie auf **Change details** (Angaben ändern), um die Optionen für den Projektnamen, die Organisationsweise Ihrer Arbeit (Agile, Scrum, CMMI), den Ort, an dem Sie Projekte hosten, und die Art und Weise anzupassen, wie Sie Ihre Arbeit mit anderen teilen.  Klicken Sie auf „Weiter“.

![VSTS-Projektdetails](_img\vs-vsts\vs-vsts-project-details.png)

5.  Die Erstellung des Kontos dauert einen Moment. Anschließend wird die VSTS-Seite für Ihr erstes Projekt mit dem von Ihnen angegebenen Namen angezeigt.  Nun können Sie mit Visual Studio Team Services loslegen!

Sie erhalten auch eine E-Mail mit der Bestätigung, dass Ihr Konto erfolgreich erstellt wurde.  Darin sind auch Ihre Konto-URL sowie die bevorzugte und die Anmelde-E-Mail-Adresse aufgeführt.  

![VSTS-Begrüßungs-E-Mail](_img\vs-vsts\vs-vsts-welcome.png)
