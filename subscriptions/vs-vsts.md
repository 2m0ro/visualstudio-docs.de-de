# <a name="visual-studio-team-services-vsts-benefit-in-visual-studio-subscriptions"></a>Visual Studio Team Services-Vorteil in Visual Studio-Abonnements

## <a name="overview"></a>Übersicht 
Kostenlose Git-Repositorys, Agile-Planungstools und gehostete Agile-Builds für jede Sprache – die perfekte Ergänzung Ihrer IDE. 
- Erstellen Sie die perfekte Entwicklungsumgebung für Ihr Team. 
- Bleiben Sie dran: von der Idee bis zum Release. 
- Verbessern Sie die Codequalität, und erkennen Sie Probleme frühzeitig.
- Automatisieren und Vereinfachen Ihrer Azure-Bereitstellungen
- Steigern Sie die Produktivität mit leistungsstarken Features.

Sie wissen nicht genau, welches Abonnement Sie verwenden?  Besuchen Sie die Website [https://my.visualstudio.com/subscriptions](https://my.visualstudio.com/subscriptions?wt.mc_id=o~msft~docs), um alle Abonnements anzuzeigen, die Ihrer E-Mail-Adresse zugewiesen sind. Wenn nicht alle Ihrer Abonnements angezeigt werden, sind möglicherweise einige Abonnements einer anderen E-Mail-Adresse zugewiesen.  Sie müssen sich mit der entsprechenden E-Mail-Adresse anmelden, um diese Abonnements anzuzeigen. 

Wenn Sie sich bei VSTS mit derselben Identität anmelden, die Sie zur Aktivierung Ihres Visual Studio-Abonnements verwendet haben, wird dies automatisch erkannt. Dies ist sowohl für die primäre Identität der Fall, die Sie zum Anmelden im Abonnentenportal verwenden, als auch für jede beliebige andere Identität, die Sie für Ihr Visual Studio-Abonnement konfiguriert haben. VSTS unterstützt sowohl Microsoft-Konten (wie @outlook.com) als auch Geschäfts-, Schul- oder Unikonten, die Azure Active Directory verwenden und von Ihrer Organisation verwaltet werden. Sie können sowohl Ihre primäre als auch Ihre anderen Identitäten in VSTS verwenden. Außerdem können Sie beliebig vielen VSTS-Konten als Mitglied beitreten.

Um VSTS zu verwenden, benötigen Sie ein Konto. Sie können dafür ein vorhandenes nutzen oder ein neues erstellen.  So erstellen Sie ein neues Konto:
1.  Melden Sie sich auf [https://my.visualstudio.com/benefits](https://my.visualstudio.com/benefits?wt.mc_id=o~msft~docs) an.
2.  Suchen Sie im Abschnitt „Tools“ die Kachel „Visual Studio Team Services“, und klicken Sie im unteren Bereich der Kachel mit den Vorteilen auf den Link „Los geht‘s“.   

Im Folgenden wird aufgeführt, welche VSTS-Komponenten in den verschiedenen Abonnements enthalten sind: 
- Visual Studio Enterprise: [Basic](https://www.visualstudio.com/team-services/compare-features/), [Test Manager](https://marketplace.visualstudio.com/items?itemName=ms.vss-testmanager-web), [Package Management](https://marketplace.visualstudio.com/items?itemName=ms.feed)
- Visual Studio Professional: [Basic](https://www.visualstudio.com/team-services/compare-features/)
- MSDN Platforms: [Basic](https://www.visualstudio.com/team-services/compare-features/), [Test Manager](https://marketplace.visualstudio.com/items?itemName=ms.vss-testmanager-web)
- Visual Studio Test Professional: [Basic](https://www.visualstudio.com/team-services/compare-features/), [Test Manager](https://marketplace.visualstudio.com/items?itemName=ms.vss-testmanager-web)

3.  Geben Sie einen Namen für Ihre VSTS-Projektwebsite ein.  

4.  Wählen Sie aus, ob Sie das Projekt mit **Git** oder **Team Foundation-Versionskontrolle (Team Foundation Version Control, TFVC)** verwalten möchten.  Dies ist eine unveränderliche Auswahl für jedes Teamprojekt, das Sie erstellen, aber Sie können TFVC und Git-Teamprojekte in derselben Teamprojektsammlung verwenden.  Sind Sie nicht sicher, wofür Sie sich entscheiden sollen? 
    - Git ist ein dezentralisiertes Quellcodeverwaltungssystem. Jeder Entwickler verfügt auf seinem Entwicklungscomputer über eine Kopie des gesamten Quellrepositorys. Entwickler können für alle Änderungen auf ihrem Entwicklungscomputer Commits ausführen und Versionskontrollvorgänge wie den Aufruf von Versionsgeschichten oder Vergleichen ohne Netzwerkverbindung vornehmen.  [Erfahren Sie mehr](https://www.visualstudio.com/docs/git/gitquickstart) über Git.
    - TFVC: Team Foundation-Versionskontrolle ist ein zentralisiertes Versionskontrollsystem. In der Regel verfügen Teammitglieder auf ihren Entwicklungscomputern nur über eine Version jeder Datei. Daten zur Versionsgeschichte einer Datei werden nur auf dem Server gespeichert. Verzweigungen sind pfadbasiert und werden auf dem Server erstellt. [Weitere Informationen](https://www.visualstudio.com/docs/tfvc/overview) zu Team Foundation-Versionskontrolle.

 
5.  Klicken Sie auf **Change details** (Angaben ändern), um die Optionen für den Projektnamen, die Organisationsweise Ihrer Arbeit (Agile, Scrum, CMMI), den Ort, an dem Sie Projekte hosten, und die Art und Weise anzupassen, wie Sie Ihre Arbeit mit anderen teilen.  Klicken Sie auf „Weiter“.

# <a name="create-your-vsts-account"></a>Erstellen eines VSTS-Kontos

6.  Die Erstellung des Kontos dauert einen Moment. Anschließend wird die VSTS-Seite für Ihr erstes Projekt mit dem von Ihnen angegebenen Namen angezeigt.  Nun können Sie mit Visual Studio Team Services loslegen!

Sie erhalten auch eine E-Mail mit der Bestätigung, dass Ihr Konto erfolgreich erstellt wurde.  Darin sind auch Ihre Konto-URL sowie die bevorzugte und die Anmelde-E-Mail-Adresse aufgeführt.
