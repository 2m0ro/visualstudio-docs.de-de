# Toolbox
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

Der **Werkzeugkasten** zeigt Symbole für Steuerelemente und andere Elemente an, die Sie [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)]\-Projekten hinzufügen können.  Zum Öffnen von **Werkzeugkasten** klicken Sie im Menü **Ansicht** auf **Werkzeugkasten**.  Sie können den **Werkzeugkasten** andocken und so festlegen, dass er entweder geöffnet angeheftet oder **Automatisch ausblenden** bleibt.  
  
 Die **Werkzeugkasten**\-Symbole können in eine Designansicht gezogen bzw. in einem Code\-Editor eingefügt werden.  Mit beiden Aktionen wird der Basiscode hinzugefügt, mit dem in der aktiven Projektdatei eine Instanz des **Werkzeugkasten**\-Elements erstellt wird.  
  
 Im **Werkzeugkasten** werden nur Elemente angezeigt, die sich für den Dateityp eignen, in dem Sie arbeiten.  Sie können eine Suche im Werkzeugkasten durchführen, um die Elemente, die angezeigt werden, zu filtern.  Wenn das Projekt ein Steuerelement erfordert, das vom Client Profile nicht unterstützt wird, können Sie durch Bearbeiten der Projekteigenschaften für das Projekt festlegen, dass das gesamte Framework als Ziel festgelegt wird.  
  
> [!NOTE]
>  Auf Ihrem Computer werden möglicherweise andere Namen oder Speicherorte für die Benutzeroberflächenelemente von Visual Studio angezeigt als die in den folgenden Anweisungen aufgeführten.  Diese Elemente sind von der jeweiligen Visual Studio\-Version und den verwendeten Einstellungen abhängig.  Weitere Informationen finden Sie unter [Customizing Development Settings in Visual Studio](http://msdn.microsoft.com/de-de/22c4debb-4e31-47a8-8f19-16f328d7dcd3).
