# Sicherheit in Visual Studio
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

Sicherheitsüberlegungen sollten vom Entwurf bis zur Bereitstellung in alle Aspekte der Anwendungsentwicklung einfließen.  Beginnen Sie damit, Visual Studio so sicher wie möglich auszuführen.  Siehe auch [Benutzerberechtigungen](../ide/user-permissions-and-visual-studio.md).  
  
 Damit Sie sichere Anwendungen effektiv entwickeln können, benötigen Sie grundlegende Kenntnisse über die Sicherheitskonzepte und Sicherheitsfeatures der Plattformen, für die Anwendungen entwickelt werden.  Außerdem sollten Kenntnisse im Bereich sicherer Codierungstechniken vorhanden sein.  
  
## Sicherheit  
 [Security](../Topic/Security%20in%20the%20.NET%20Framework.md)  
 Beschreibt Codezugriffssicherheit, rollenbasierte Sicherheit, Sicherheitsrichtlinien und Sicherheitstools in .NET Framework.  
  
 [Die zehn wichtigsten Tipps zum Schützen des Codes, die jeder Entwickler beachten sollte](http://go.microsoft.com/fwlink/?LinkId=72877)  
 Beschreibt die wichtigsten Punkte, die Sie in Betracht ziehen sollten, damit Ihre Daten im System nicht gefährdet sind.  
  
## Richtige Codierung zur Gewährleistung der Sicherheit  
 Die meisten Codierungsfehler, die Sicherheitsmängel verursachen, entstehen, weil Entwickler bei Benutzereingaben von falschen Voraussetzungen ausgehen oder nicht vollständig mit der Plattform vertraut sind, für welche die Anwendung entwickelt wird.  
  
 [Secure Coding Guidelines](../Topic/Secure%20Coding%20Guidelines.md)  
 Stellt Richtlinien zum Klassifizieren der Komponenten in Bezug auf die Sicherheitsprobleme bereit.  
  
 [Empfohlene Vorgehensweisen bezüglich der Sicherheit](/visual-cpp/top/security-best-practices-for-cpp)  
 Behandelt Pufferüberläufe und beschreibt ausführlich das über den Compilerschalter \/GS bereitgestellte Sicherheitsüberprüfungsfeature von Microsoft Visual C\+\+.
