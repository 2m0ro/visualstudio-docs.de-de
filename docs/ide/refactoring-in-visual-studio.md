# Umgestaltung in Visual Studio
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

Umgestaltung stellt den Prozess der Codeänderung mit dem Ziel dar, diesen leichter verwalten, verstehen und erweitern zu können, ohne dabei das Verhalten zu ändern.  Einige häufige *Umgestaltungen* umfassen Folgendes:  
  
1.  Aufteilen von langen Funktionen in kleinere Funktionen  
  
2.  Umbenennen von Variablen, Typen, Funktionen oder Parametern  
  
3.  Neuanordnen von Parametern  
  
4.  Entfernen oder Neuanordnen von using\-Direktiven  
  
 Da die Logik der Umgestaltung eng mit der Sprachsyntax verbunden ist, stellt jeder Sprachdienst in Visual Studio eigene Umgestaltungsfunktionen bereit.  Weitere Informationen finden Sie auf den folgenden Seiten:  
  
### Umgestaltung in C\#  
 [Umgestaltung \(C\#\)](../csharp-ide/refactoring-csharp.md)  
  
### Umgestaltung in Visual Basic  
 [Umgestaltung und Dialogfeld "Umbenennen" \(Visual Basic\)](/dotnet/visual-basic/developing-apps/using-ide/refactoring-and-rename-dialog-box)  
  
### Umgestaltung in C\+\+  
 [Schreiben und Umgestalten von Code \(C\+\+\)](/visual-cpp/ide/writing-and-refactoring-code-cpp)  
  
### Umgestaltung in F\#
