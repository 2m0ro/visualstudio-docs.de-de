# Projektarchitektur Typen
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

Dieser Abschnitt enthält ausführliche Informationen über die Architektur von Projekttypen in [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)].  
  
## In diesem Abschnitt  
 [Elemente eines Projektmodells](../../extensibility/internals/elements-of-a-project-model.md)  
 Führt die Dienste auf, die einen Projekttyp verwenden kann und Schnittstellen, die implementiert werden muss.  
  
 [Projekt\-Modell\-Kernkomponenten](../../extensibility/internals/project-model-core-components.md)  
 Beschreibt die Schnittstellen implementieren müssen, wenn sie beide Projekttypen und optional implementieren können, um zusätzliche Funktionen bereitzustellen.  
  
 [Erstellen von Projekttypen](../../extensibility/internals/when-to-create-project-types.md)  
 Unterstützt entscheiden Sie, wenn Sie einen Projekttyp erstellen müssen, und wenn Sie eine andere [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] VSPackages und Editoren wie das Feature für Erweiterungen verwenden können, um dasselbe Ziel zu erreichen.  
  
 [Hierarchien und Auswahl](../../extensibility/internals/hierarchies-and-selection.md)  
 Beschreibt, wie [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] Hierarchien und Auswahlkontext verwendet, um eine konsistente und vereinfachte Benutzerfreundlichkeit bereitgestellt.
