# &quot;Bearbeiten und Fortfahren&quot;
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

Mit Bearbeiten und Fortfahren sparen Sie Zeit, weil Sie damit Änderungen am Quellcode vornehmen können, während sich ein Programm im Unterbrechungsmodus befindet.  Wenn Sie die Ausführung des Programms fortsetzen, z. B. durch einen Ausführungsbefehl wie **Continue** oder **Step**, wendet Bearbeiten und Fortsetzen die Codeänderungen mit einigen Einschränkungen automatisch an.  Auf diese Weise können Sie den Code während einer Debugsitzung ändern, ohne das Programm anzuhalten, komplett neu zu kompilieren und anschließend eine neue Debugsitzung zu starten.
