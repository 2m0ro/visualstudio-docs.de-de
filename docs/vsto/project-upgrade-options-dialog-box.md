# <a name="project-upgrade-options-dialog-box"></a>Projekt aktualisieren, Optionen (Dialogfeld)
  Auf der Seite **Projekt-Upgrade** geben Sie an, ob Visual Studio automatisch VSTO-Add-In-Projekte für die Version von Office ändert, die auf dem Entwicklungscomputer installiert ist.  
  
 Um auf diese Seite zuzugreifen, klicken Sie im Menü **Extras** auf **Optionen**. Erweitern Sie den Ordner **Office-Tools** , und klicken Sie dann auf **Projekt-Upgrade**.  
  
## <a name="uielement-list"></a>UIElement-Liste  
 **Immer Upgrade auf installierte Version von Office durchführen**  
 Diese Option wird für [!INCLUDE[vs_dev12](../vsto/includes/vs-dev12-md.md)] deaktiviert, da Office-Projekte in [!INCLUDE[vs_dev12](../vsto/includes/vs-dev12-md.md)] nur [!INCLUDE[office14_long](../vsto/includes/office14-long-md.md)] - oder Microsoft [!INCLUDE[Office_15_short](../vsto/includes/office-15-short-md.md)] -Anwendungen zum Ziel haben können. Weitere Informationen finden Sie unter [aktualisieren und Migrieren von Office-Projektmappen](../vsto/upgrading-and-migrating-office-solutions.md).
