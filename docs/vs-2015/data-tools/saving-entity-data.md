# <a name="saving-entity-data"></a>Speichern von Entitätsdaten
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Wenn Sie Entitäten erstellen, aktualisieren oder löschen, die über WCF Data Services eine Verbindung mit einer Datenbank herstellen, müssen sie die Änderungen explizit in der Datenbank speichern. In den folgenden Themen wird beschrieben, wie Änderungen an Entitätsdaten in WCF-Data Services-Anwendungen gespeichert werden.  
  
|Thema|Beschreibung|  
|-----------|-----------------|  
|[Vorgehensweise: Hinzufügen, Ändern und Löschen von Entitäten](http://msdn.microsoft.com/library/a00f8933-b232-4445-95ba-adc634f055d8)|Zeigt, wie Änderungen an Entitätsdaten über einen Datendienst gespeichert werden.|  
|[Vorgehensweise: Ausführen von asynchronen Datendienstabfragen](http://msdn.microsoft.com/library/902a2dc1-d0e9-4b00-90a8-becc4cb1f6a7)|Zeigt, wie über einen Datendienst Änderungen asynchron gespeichert werden.|
