# <a name="developing-workflow-applications-targeting-the-net-30-or-net-35-framework"></a>Entwickeln von Workflowanwendungen, die auf .NET 3.0 oder .NET auf 3.5 Framework abzielen

Visual Studio 2010 enthält die Windows-Workflow-Designer zum Erstellen von Windows Workflow Foundation (WF)-Anwendungen, die auf .NET Framework 4 abzielen. Der Designer verwendet die vertraute Visual Studio-Benutzeroberfläche für die grafische Erstellung und Debuggen von workflowanwendungen.

Visual Studio 2010 bietet außerdem Zugriff auf einen legacyworkflow-Designer, der .NET Framework, Version 3.5 oder die WinFX als Ziel verwendet werden kann. Dieser Abschnitt enthält Anweisungen zur Verwenden dieses Legacydesigners.
