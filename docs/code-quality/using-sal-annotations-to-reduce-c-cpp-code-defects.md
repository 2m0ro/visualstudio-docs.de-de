# Verwenden von SAL-Anmerkungen zum Reduzieren von C/C++-Codefehlern
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

SAL ist die Microsoft\-Quellcodeanmerkungssprache.  Indem Sie Quellcodeanmerkungen verwenden, können Sie die Absicht hinter dem Code explizit machen.  Diese Anmerkungen können auch automatisierte Tools der statischen Analyse, um den Code, mit erheblich weniger falschen Positiven und False Negativen genauer zu analysieren.  
  
 Die Artikel in diesem Abschnitt der Dokumentation werden Aspekte des SALZES, unterstellen Verweis für SALZsyntax fest und nennen Beispiele der Verwendung.  
  
-   [Einführung in SAL](../code-quality/understanding-sal.md)  
  
     Enthält Informationen und Beispiele, die die SAL\-Anmerkungen Kern anzeigen.  
  
-   [Hinzufügen einer Anmerkung zu Funktionsparametern und Rückgabewerten](../code-quality/annotating-function-parameters-and-return-values.md)  
  
     Führt die SAL\-Anmerkungen für Funktionsparameter und Funktionen auf.  
  
-   [Hinzufügen einer Anmerkung zum Funktionsverhalten](../code-quality/annotating-function-behavior.md)  
  
     Führt die SAL\-Anmerkungen für Funktionen und Funktionsverhalten auf.  
  
-   [Hinzufügen einer Anmerkung zu Strukturen und Klassen](../code-quality/annotating-structs-and-classes.md)  
  
     Führt die SAL\-Anmerkungen für Strukturen und Klassen auf.  
  
-   [Hinzufügen einer Anmerkung zum Sperrverhalten](../code-quality/annotating-locking-behavior.md)  
  
     Erläutert, wie mit Sperrenmechanismen SAL\-Anmerkungen verwendet.  
  
-   [Angeben, wann und wo eine Anmerkung gültig ist](../code-quality/specifying-when-and-where-an-annotation-applies.md)  
  
     Führt die SAL\-Anmerkungen auf, die der Zustand oder den Bereich \(Platzierung\) anderer SAL\-Anmerkungen angeben.  
  
-   [Systeminterne Funktionen](../code-quality/intrinsic-functions.md)  
  
     Führt die systeminternen SAL\-Anmerkungen auf.  
  
-   [Empfohlene Vorgehensweisen und Beispiele](../code-quality/best-practices-and-examples-sal.md)  
  
     Enthält Beispiele, die anzeigen, wie SAL\-Anmerkungen verwendet.  Erläutert auch allgemeine Fehler.
