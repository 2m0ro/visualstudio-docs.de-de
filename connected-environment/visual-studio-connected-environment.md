# <a name="vs-connected-environment-for-azure-container-service-aks-preview"></a>Visual Studio Connected Environment für Azure Container Service (AKS) (Vorschau)
Visual Studio Connected Environment unterstützt Sie dabei, in Kubernetes schnellere Entwicklungen durchzuführen. Erstellen Sie eine auf Kubernetes basierende Umgebung, die vollständig in Azure verwaltet wird und für die Entwicklung optimiert ist, und entwickeln Sie dann iterativ Container in der Cloud mithilfe von vertrauten Tools wie Visual Studio Code, Visual Studio oder der Befehlszeile.

Diese Vorgehensweise bringt einige Vorteile mit sich:

* Sie erhalten eine Entwicklungsumgebung ohne Infrastruktur, die repräsentativ für die Produktion mit vollem Zugriff auf Cloudressourcen ist.
* Sie können Node.js- und .NET Core-Container direkt in Kubernetes mit Visual Studio Code oder Visual Studio debuggen. Alle anderen Programmiersprachen können in der Befehlszeilenschnittstelle für die Entwicklung verwendet werden.
* Sie können eine Kubernetes-Instanz für Ihr Entwicklungsteam freigeben, um Kosten zu sparen und den Aufwand für das Einrichten von lokalen Computern für neue Teammitglieder zu minimieren.
* Sie können Ihren Code isoliert entwickeln und End-to-End-Tests mit anderen Komponenten durchführen, ohne Abhängigkeiten replizieren oder imitieren zu müssen.
